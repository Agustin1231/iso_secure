01 Descripción del Proyecto

ISO_SECURE es una API REST desarrollada con FastAPI que expone endpoints para consultar, registrar y analizar los Indicadores Clave de Rendimiento (KPIs) del Sistema de Gestión de Seguridad de la Información (SGSI) de una organización, alineados con la norma ISO/IEC 27001:2022 y el marco de medición ISO/IEC 27004.

El sistema permite a la alta dirección disponer de información cuantitativa y trazable sobre el estado de seguridad de la información, facilitando la toma de decisiones, la mejora continua bajo el ciclo PHVA (Planificar-Hacer-Verificar-Actuar) y el cumplimiento normativo.

04 Endpoints de la API

/incidents — Gestión de Incidentes de Seguridad

Módulo para el registro, seguimiento y análisis de incidentes de seguridad de la información.

Método Endpoint Descripción
GET /incidents Listar todos los incidentes de seguridad registrados. Soporta filtros por estado (open, in_progress, closed), severidad y rango de fechas.
POST /incidents Reportar un nuevo incidente de seguridad. Requiere título, descripción, categoría y severidad (low, medium, high, critical).
GET /incidents/{id} Obtener el detalle completo de un incidente específico por su UUID, incluyendo historial de cambios de estado.
PUT /incidents/{id}/status Actualizar el estado de un incidente (open → in_progress → closed). Registra automáticamente la fecha de resolución.
GET /incidents/stats Obtener KPIs de incidentes: total reportados, tasa de resolución (%), tiempo medio de resolución (MTTR) en horas.

/controls — Controles ISO 27001

Gestión de los controles de seguridad del Anexo A de ISO 27001, incluyendo su estado de cumplimiento.

Método Endpoint Descripción
GET /controls Listar todos los controles ISO 27001 implementados. Filtrable por dominio (A.5, A.6, etc.) y estado de cumplimiento.
POST /controls Registrar un nuevo control de seguridad con su referencia ISO, nombre, descripción, responsable y porcentaje de cumplimiento inicial.
PUT /controls/{id} Actualizar el estado de cumplimiento de un control (compliant, partial, non_compliant) y su porcentaje de implementación (0-100%).
GET /controls/compliance Obtener el porcentaje global de cumplimiento de controles ISO 27001, desglosado por dominio del Anexo A.

/risk — Gestión de Riesgos

Evaluación y monitoreo del nivel de riesgo del SGSI según la metodología de probabilidad × impacto.

Método Endpoint Descripción
GET /risk/current Obtener el nivel de riesgo actual consolidado del SGSI, incluyendo score global (1-25) y nivel cualitativo (low, medium, high, critical).
POST /risk Registrar una nueva evaluación de riesgo por dominio. Requiere probabilidad (1-5), impacto (1-5) y dominio ISO 27001.
GET /risk/history Consultar el historial de evaluaciones de riesgo por período. Permite analizar la evolución temporal del riesgo.
GET /risk/by-domain Obtener el mapa de riesgo desglosado por cada dominio del Anexo A de ISO 27001 (A.5 a A.18).

/dashboard — Panel Ejecutivo

Endpoints para el dashboard de la alta dirección con visión consolidada de todos los KPIs del SGSI.

Método Endpoint Descripción
GET /dashboard/summary Resumen ejecutivo consolidado: incidentes activos, cumplimiento global, nivel de riesgo, dominio más crítico. Ideal para revisiones por la dirección (ISO 27001 Cláusula 9.3).
GET /dashboard/kpis Todos los KPIs detallados con su valor actual, tendencia, umbral de alerta (verde/ámbar/rojo) y descripción según ISO 27004.
GET /dashboard/export Exportar todos los datos del dashboard en formato JSON o CSV para reportes externos o integración con otras herramientas.

07 Alineación con ISO/IEC 27001:2022

El diseño de la API responde directamente a la Cláusula 9 de ISO 27001 (Evaluación del Desempeño), la cual exige que la organización evalúe el rendimiento y la eficacia del SGSI de manera sistemática.

Norma / Cláusula Aplicación en la API
ISO 27001 — Cláusula 9.1 Los endpoints /incidents/stats y /controls/compliance proveen las métricas de desempeño
ISO 27001 — Cláusula 9.3 /dashboard/summary genera el reporte ejecutivo para la alta dirección
ISO 27004 Cada KPI tiene definida su fórmula, fuente de datos y umbral de alerta
ISO 27005 El módulo /risk implementa la matriz probabilidad x impacto
